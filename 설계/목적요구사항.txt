- 목적

	공연/이벤트 등 좌석 기반 티켓팅을 위한 시스템 설계 및 구현

	수많은 사용자 동시 접근을 처리할 수 있는 고성능 & 고신뢰 백엔드 아키텍처 구축

	분산 락, 예약 만료, 인당 제한, 좌석 상태 캐싱, 대기열 시스템 등 핵심 기능 구현

- 요구사항

 1. 좌석 단위 예매 기능

	공연의 좌석 정보를 기반으로 티켓 생성
	
	좌석 단위로 상태 관리: AVAILABLE, PENDING, RESERVED, CANCELLED

 2. 사용자 예매 흐름
	
	특정 좌석을 선택 → 일정 시간 내 결제 → 좌석 확정

	결제 실패 또는 시간 초과 시 좌석 자동 복귀

 3. 예매 정책
	
	인당 예매 가능 수 제한 (limitPerUser)

	예매 오픈 시간(openAt) 설정

	예매 방식(OpenType)은 확장 가능

 4. 대기열

	redis : Zset 사용

- 기능 요소
	
	동시성 제어		Redisson 분산 락을 활용해 좌석 중복 선택 방지
	예매 만료 처리		PENDING 상태가 일정 시간 초과 시 자동 취소 (Scheduled Task)
	좌석 상태 캐싱		Redis를 이용해 실시간 좌석 현황을 빠르게 제공
	인당 제한 정책		사용자별 예매 수 제한 로직 구현
	대기열 처리		Redis SortedSet을 이용한 대기열 큐 (예정)
	부하 견고성 설계		분산 락, 캐시 일관성, 예약 워커 등으로 트래픽 감내

- 추가 기능
	
	대기열 Polling API 구현 (/queue/status)

 	부하 테스트 (k6 or Locust) 통한 병목 지점 확인 및 개선

 	WebSocket 없이 polling 방식으로 실시간성 유사 구현

 	관리자 페이지: 좌석 현황, 예매자 통계, 회차별 설정 등
